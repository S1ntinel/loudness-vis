## demo 0.5

- 版本代号：`demo 0.5`
- 记录日期：`2026-05-14`
- 版本定位：从"分析仪表盘"升级为"分析 + MV 创作工作站"，并完成中英文双语切换

### 版本摘要

`demo 0.2` 把分析做透，`demo 0.5` 把 **MV 编辑器**做出最小可用版本，同时给整套界面铺上**国际化（i18n）**底座，开始对外英文用户开放。0.3 / 0.4 阶段的迭代（色彩预设系统、SettingsPanel、性能优化、设备布局重构等）全部合并进本版本一次性发布。

主要方向：

- **新增 MV Tab**：5 模板 × 5 主题 × 8 槽位效果系统，支持多素材导入（音频 / 视频背景 / 封面图 / 歌词 / 字体）+ 卡拉 OK 逐字歌词 + 5 个质量预设的视频导出（B 站 1080p / YouTube 1080p / 4K / 抖音竖屏 / 归档）
- **全局中英文切换**：新增右上角 🌐 切换按钮，覆盖工具栏、4 个 Tab、设置面板、MV / 设备 / 录音 / 分析所有子面板 (~300 条文案)，选择持久化到 localStorage
- **设备 Tab 重构**：从 2×2 网格 → 上下流式 6 section（输入 / 输入监控 / 输出 / 输出路由 / Windows 合成器 / 系统信息），设备卡紧凑化（高度 −30%），监控面板抽出为独立卡
- **频谱图配色修复**：5 个调色板（Magma / Viridis / Plasma / Inferno / Cool）切换按钮原本不生效，bug 修复
- **色彩主题预设系统**：默认 / 绿色 / 粉色 三套配色，左侧 PresetBar 紧凑切换 + 设置面板大卡切换
- **荧光勾线 + 散点视觉强化**：分析仪表盘所有 panel 复刻 FoRk 视觉风格（频谱图、频响曲线、Goniometer 散点、LUFS 条形指示等）
- **增益滑块独立于音量**：±12 dB，与主音量分离
- **后半段播放卡顿修复**：去掉逐像素 `shadowBlur` 累积，4 分钟歌曲拉到尾部不再延迟

### 这一版的新功能

#### MV 编辑器（新 Tab）

- **5 模板 × 5 主题 × 8 槽位**：背景 / 频谱 / 粒子 / 文字 / 视频背景 / 图片层 / 卡拉 OK 歌词 自由组合
- **素材库**：5 类素材分组（音频 / 视频背景 / 封面图 / 歌词 / 字体），单选激活，支持拖拽和点击添加
- **视频背景**：mp4 / webm / mov，`drawImage` 每帧绘制到 Stage canvas，支持 cover / contain / stretch 三种填充
- **图片层**：png / jpg / webp / avif，独立的缩放 / 偏移 / 模糊参数
- **卡拉 OK 歌词**：支持 `.lrc` 行级 + enhanced `.lrc` 字级（行内 `<00:00.00>` 标记）+ `.t.lrc` 翻译对照；当前行字级渐变 mask 高亮，上下两行半透明，翻译堆叠在原文下方
- **自定义字体**：ttf / otf / woff / woff2，`FontFace API` 注册到 `document.fonts`，文字层和歌词自动应用
- **项目导入导出**：三种模式 — `path`（仅路径）/ `inline`（base64 内嵌，单文件分享）/ `zip`（`.mvproj.zip` 含完整 assets/，跨机迁移）
- **5 个视频导出预设**：B 站 1080p (8 Mbps) / YouTube 1080p (12 Mbps) / YouTube 4K (35 Mbps) / 抖音竖屏 1080×1920 (6 Mbps) / 归档 (20 Mbps)；非归档预设用 offscreen canvas 缩放到目标分辨率（保持比例 + 黑边）

#### 全局国际化（i18n）

- 接入 `react-i18next`，资源以 JSON 形式分中英两份
- 顶栏新增 🌐 切换按钮（设置按钮右侧），点击在 `中` / `EN` 间切换，localStorage 持久化
- 中文 / 英文两版各约 300 条文案，覆盖工具栏、4 个 Tab、所有面板、错误提示、按钮 tooltip
- 错误消息支持插值（如 `winSessionFailFmt: "无法读取 Windows 音频会话：{{msg}}"`）
- MV 主题名 / 模板名 / 描述 / 频谱样式 / 背景类型 / 导出预设 / 8 个特效名 / 5 类素材类型都翻译

#### 设备 Tab 布局重构

- 2×2 grid → 上下流式 6 section（按工作流顺序排列）
- 设备卡 padding 12px → 8px，字号 14 → 13，隐藏冗余的 deviceId（移到 hover title），整体高度 −30%
- 输入监控 / 输出路由从设备卡内嵌抽出为独立 section，仅在选中设备时显示

#### 顶部工具栏与设置

- **色彩主题预设**：左侧 `< Default ●●● >` 紧凑切换器（圆点 + 上下箭头）
- **SettingsPanel 弹窗**：⚙️ 设置按钮唤起，含外观 / 音频 / 布局三个 Tab；外观区可切深 / 浅 + 三套预设大卡
- **增益滑块独立显示**：工具栏右侧，−12 ~ +12 dB，与音量分离
- **置顶图钉**：Electron 环境下保留

#### 频谱图修复 + 分析仪表盘视觉强化

- **5 个 colormap 切换按钮修复**：之前 `useEffect` 空依赖 + 没把 `colorMap` 算作 bitmap invalidation 条件，导致切换无效；现在改用 ref + lastColorMapRef 比较，5 个配色都可见
- **荧光勾线**：频响曲线、波形已播放区域、Goniometer 散点、Spectrum stroke 都加 shadow glow
- **指标卡片底部光带**：每个 stat 卡片加随主题色变化的发光底纹
- **频响曲线 10k+ 优化**：暂停时冻结最后一次有效数据，避免无声音段抖动

#### 性能修复

- **后半段播放卡顿**：原来 Waveform 三层 stroke 中每像素都触发 `shadowBlur`，4 分钟歌拉到尾部 6000+ 次模糊调用堆积 → 改成单层 stroke + 整段 `globalCompositeOperation = 'lighter'` 一次性 glow，性能提升数倍

### 已知边界

- MV Tab 中用户在效果机架手动切换 slot 类型后，若再触发 `applyTemplate`/`applyTheme` 会覆盖手动修改（FoRk MV 原设计，PR-5 范围内未重构）
- 音频素材原始字节在导出时无法保留（已被 `decodeAudioData` 消费成 `AudioBuffer`），zip / inline 模式自动降级为 path 引用；用户需要把音频文件单独放在 .mvproj.zip 同级目录
- flac 解码依赖浏览器，Chrome 实测不支持；UI 已在上传失败时给出"先转 wav"提示
- `systemMixerError` 等已持有的错误字符串在切换语言后不会自动重新翻译（需要重新触发错误才更新）
- MV 项目录制 4K 时码率受 MediaRecorder + GPU 限制，部分机器可能掉帧；当前 8 Mbps / 12 Mbps 预设最稳定

### 这一版的新功能

#### 分析 Tab

- **波形多频段染色**支持三种模式（单色 / RGB 三频段叠加 / 频谱重心 hue 映射），后两种基于 STFT 计算，对响度战争音频也能拉出可读的色差
- **LUFS 数显从 2 项扩到 5 项**：Momentary（400ms）/ Short-term（3s）/ Integrated（整曲，EBU R128）/ True Peak（4× 上采样后的 dBTP）/ Loudness Range
- **频响曲线支持多轨叠加对比**：标题栏的「+ 加入对比」按钮，每条对比通道独立颜色 + 半透明描边 + 可隐藏/移除
- **新增频谱图面板**：整曲 STFT 时频热图，Magma 调色板，时间-频率二维呈现
- **频谱图加准星和拖动播放头**：与频响曲线一致的十字浮窗（显示时间 / 频率 / dB），与波形面板一致的拖拽进度条 + 红色光晕
- **波形 / 频谱图时间轴同步缩放**：Shift + 滚轮以鼠标位置为锚点放大；两个面板同步；双击重置
- **波形 / 频谱图之间加可拖拽分隔条**：自由调整两面板高度比例，比例记忆到 localStorage
- **声场指示器加"球面"模式**：俯视图显示头部 + 4 频段彩色球（低 / 中低 / 中高 / 高），球的位置由实时 L/R 平移决定，大小由能量决定，原 Goniometer 散点保留作为切换项

#### 录音 Tab

- **统一为多轨道编辑器**：录音和上传合并为 `Track` 数据结构（含 audioBuffer / peaks / trim 范围 / 选中状态）
- **拖入和工具栏选文件 → 弹 Modal 选目标**（送入分析 / 送入轨道）；录音 Tab 内的「+ 上传音频」按钮不弹 Modal，直接入轨
- **每条轨道带迷你波形 + 双端点截取**：左右黄色把手拖端点、中间区域拖整段、双击重置；导出片段会按 trim 范围切片成 WAV
- **试听片段 toggle**：单击「▶ 试听 / ⏸ 暂停 / ▶ 继续」，复用一个全局播放器
- **多轨混音**：左侧复选框勾选 N 条轨道，列表头「▶ 播放选中 (n)」一键混音，同样支持暂停/继续，或用「■ 停止」结束
- **录音区比例调整**：上方录音控制压缩到 180px，下方轨道列表吃满剩余空间

#### 顶部工具栏

- **Electron 窗口置顶图钉**：📌 按钮，调用 `BrowserWindow.setAlwaysOnTop`，浏览器环境自动隐藏（依赖 `window.electronAPI` 探测）
- **深 / 浅主题切换**保留，状态记忆到 localStorage

### 当前交互决策记录

延续 demo 0.1 的方向，本版本中确定的交互细节包括：

- 上传文件后弹 Modal 让用户选「分析 / 轨道」，避免与现有"拖入加载到分析"动作冲突
- 双端点截取：端点能拖、中间也能拖整段，双击波形重置
- 试听用独立按钮 toggle，不与拖动播放头共用同一手势
- Shift + 滚轮缩放，锚点为鼠标位置，避免和正常滚动冲突
- 染色 RGB 模式直接把高/中/低频段对应到 R/G/B 通道，让"既有低频又有高频"的瞬间可读
- LUFS 显示横排 5 项，避免占用过多垂直空间

### 当前版本已知边界

- 阶段 5（系统设备 + 应用音量控制）尚未开始，依赖 Electron 主进程接 Windows COM API
- 阶段 7（MV 编辑器 Tab）尚未开始，预计是独立路由 + Canvas 编辑栈 + 视频导出，规模较大
- 多轨道目前只在录音 Tab 编辑/试听，分析 Tab 的多曲线对比仍以静态平均频谱形式呈现，未做"播放期间多轨实时分析"
- True Peak 是 4× 线性上采样的近似版本，不是 sinc 插值，跟专业母带工具会有 ±0.5 dB 偏差
- 频谱图分辨率固定（1500 时间帧 × 512 频段 bin），后续可考虑做"按缩放级别动态加细"
- 频谱图准星显示的 dB 是 `dbFloor + v/255 × (dbCeil - dbFloor)` 反映射，跟实际 STFT 输出的 dB 有量化误差
